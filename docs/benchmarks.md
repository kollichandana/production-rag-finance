
A table comparing the **same questions** under both pipelines will land here, plus a JSON dump of per-item results under `data/eval/latest_report.json`.

## Why we publish this

A RAG project without measured quality is a vibes project. Recruiters and clients ask "how do you know it's good?" — this is the answer.

The naive baseline is the same code as production with every toggle flipped off:
- No router, rewriter, HyDE, decomposition
- No reranker
- No compression, no reorder
- No grounding check
- No semantic cache

Everything else (model, embeddings, vector store, corpus, benchmark questions) is identical. The delta is purely the pipeline.
